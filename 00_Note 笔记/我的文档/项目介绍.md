### 某机构数据爬取搜索

那段时间在考软考高项，系统分析师，我当时是报了机构的，这样提高一点学习效率，节省一点时间嘛，机构提供课程和题库，但是机构的这个题库有些功能不是很完善，也可能是怕题库泄漏这类的问题吧，不支持关键词搜索。后续临近考试复习的时候，很难总结，所以我就干了这么一件事情，我把题库爬了下来，把一些文档资料爬取了下来。

我先说我已经做过的，再说我没做项计划去做的事情哈

1. Python + Selenium 技术爬取到题库、资料总所有信息，写入 MySQL 数据库中，这个数据库是我个人基于题库去设计的关联模式
	1. 试卷表：试卷类型、查看机构题库的链接
	2. 题目表：关联试卷表，题目内容，这道题的解析
	3. 选项表：关联题目表，选项和题目是 多对一关系，正确答案
	4. 图片表：图片ID、图片URL
	5. 图片与题目关联表：标识（题目图片、解析图片），题目ID 和 图片ID关联，解决多对多关系问题。
2. SpringBoot + MyBatisFlex 做数据接口，我需要搭建一个前端页面，供我查询，我比较追求完美，不屑于直接在数据库里做 like 操作，而且关联起来，属实有点不太好用，也是处于对技术追求，
	1. 使用 LogState 将 MySQL 数据同步到 ElasticSearch，es 的作用就是他对分词查询有天然优势，通过使用 `RestHighLevelClient` 我可以很轻松的查询到数据，甚至给查询词一个高亮。
	2. 这里用 Mybatis-Flex 的原因是，这个东西其实和 MyBatis-Plus 在基础的 CRUD 接口上没啥太大区别，所以不需要特地去学习，但是他独有一些 读写分离、动态表名这些技术，我看过一些技术 Up 主对他的使用，所以特地用这个新技术去尝试，去学习。可能现在我为了快速实现这个题库搜索，不会去特地用这些技术，但是保不齐以后哪天兴趣来了，就可以用这个项目做一个技术性验证。
3. Vite + Vue3 + TS
	1. 前端的东西真的特别特别多，我以前觉得前端可能就 Vue 啊 JS、CSS 用的好一点，前端就差不多了，其实非常非常多内容。
	2. Pinia、Sass、Mock、Prettier、commitlint 等
